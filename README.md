## 面接官向け テスト方法 (簡易版)

無料トライアルを開始するボタンからダッシュボードへ移動します。

### 1. スクレイピング機能

* Wantedlyの求人URLを入力し、データが正しく表示されるか確認してください。
    * **テスト用URL:**
        ```
        https://www.wantedly.com/projects?new=true&page=1&occupationTypes=jp__engineering&hiringTypes=internship&areas=tokyo&order=mixed
        ```
    * **確認ポイント:**
        * 入力したURLの求人情報がテーブル形式で表示されるか。
        * 必要な情報（企業名、職種、募集要項など）が正しく取得できているか。

### 2. 応募企業管理 (CRUD機能)

* 企業情報の登録、編集、削除機能が正常に動作するか確認してください。
    * **確認ポイント:**
        * 新規企業情報を登録できるか（必須項目が適切にバリデーションされるか）。
        * 登録した企業情報を編集できるか。
        * 登録した企業情報を削除できるか。

### 3. 選考結果分析

* 選考結果を複数登録し、グラフが正しく表示されるか確認してください。
    * **確認ポイント:**
        * 選考結果（書類選考、一次面接、最終面接など）を登録できるか。
        * 登録した選考結果がグラフ（例: 棒グラフ）で表示されるか。
        * グラフの表示が登録したデータと一致しているか（件数、日付など）。

### 4. レスポンシブデザイン

* PCとスマートフォンなど異なる画面サイズで表示し、レイアウトが崩れないか確認してください。
    * **確認ポイント:**
        * 主要な画面要素（ナビゲーション、テーブル、グラフなど）が適切に配置されているか。
        * 横スクロールが発生しないか（特にスマートフォン表示）。
        * 画面サイズを変更した際に、レイアウトがスムーズに調整されるか。

### 5. APIテスト

* PostmanなどのAPIクライアントを使用して、以下のAPIが正常に動作するか確認してください。
    * **スクレイピングAPI:** 求人URLをPOSTしてデータを取得できるか。
    * **企業情報API:**
        * 企業情報をPOSTして登録できるか。
        * 特定の企業情報をGETで取得できるか。
        * 企業情報をPUTで更新できるか。
        * 企業情報をDELETEで削除できるか。
    * **確認ポイント:**
        * APIリクエストが成功（ステータスコード200など）するか。
        * APIのレスポンスが期待通りのデータ形式（JSONなど）であるか。

### 6. エラーハンドリング

* 無効なURLや不正なデータを入力し、エラーメッセージが適切に表示されるか確認してください。
    * **確認ポイント:**
        * スクレイピング機能で無効なURLを入力した際に、分かりやすいエラーメッセージが表示されるか。
        * 企業情報登録・編集画面で必須項目が未入力の場合や、不正な形式のデータを入力した場合に、適切なエラーメッセージが表示されるか。

# 概要

Wantedlyで見つけた長期インターンシップの選考プロセスを効率的に管理するためのWebアプリケーションです。求人情報のスクレイピング、応募企業管理、選考結果分析などの機能を提供し、インターンシップ探しを包括的にサポートします。

## 目的

* Wantedlyの求人情報を一元管理し、効率的なインターンシップ探しを支援します。
* 応募企業の選考状況や面接メモを記録することで、選考結果の分析を容易にします。
* 地域ごとの応募状況を可視化することで、戦略的な応募をサポートします。

## ターゲットユーザー

* エンジニアの長期インターンシップを探している大学生
* 効率的なインターンシップ選考プロセスの管理を求めている方
* データ分析に基づいて戦略的にインターンシップを探したい方

## 主な機能

* **求人情報スクレイピング:**
    * Wantedlyの求人情報を定期的にスクレイピングし、データベースに保存します。
    * スクレイピング結果をテーブル形式で表示し、分析を可能にします。
    * 日ごとの求人数をグラフで表示し、市場動向を可視化します。
* **応募企業管理 (CRUD):**
    * 応募した企業情報の登録、編集、削除（CRUD操作）が可能です。
    * 選考結果、面接メモ、地域情報などを記録できます。
* **選考結果分析:**
    * 登録した選考結果を可視化し、不採用の原因分析などに役立てられます。
    * 地域ごとの応募企業数を管理できます。
* **レスポンシブデザイン:**
    * PC、タブレット、スマートフォンなど、様々なデバイスで快適に利用できます。

## 技術スタック

* フロントエンド: Next.js (TypeScript), Tailwind CSS, shadcn-ui
* バックエンド: Django REST Framework (Python)
* データベース: SQLite (開発環境), PostgreSQL (本番環境)
* APIテスト: Postman
* デプロイ: Render, Vercel
* その他: Git, GitHub, Docker
* スクレイピング: Beautiful Soup, requests
* グラフ作成: chart.js, react-chartjs-2




## 開発背景

エンジニアの長期インターンシップ選考の競争が激化しており、選考結果の記録・分析の重要性が増しています。また、地域ごとの応募企業管理が煩雑であることや、Wantedlyの求人情報分析に手間がかかるという課題を解決するために開発しました。

## 工夫した点

* **スクレイピング機能:**
    * 定期的なスクレイピングにより、常に最新の求人情報を取得します。
    * 求人数の推移をグラフで表示することで、市場動向を視覚的に把握できます。
* **応募企業管理:**
    * CRUD操作により、柔軟な情報管理が可能です。
    * 面接メモや選考結果を記録することで、分析を容易にします。
* **UI/UX:**
    * Tailwind CSSとshadcn-uiを使用し、モダンで洗練されたUIを実現しました。
    * 直感的な操作が可能なシンプルなデザインを心がけました。
    * レスポンシブデザインにより、様々なデバイスで快適に利用できます。
* **データ処理:**
    * スクレイピングしたデータを効率的にデータベースに保存します。
    * [pandas](https://pandas.pydata.org/)を用いてデータ分析を行います。
    * chart.jsとreact-chartjs-2を用いて、データを視覚的に表示します。
* **セキュリティ:**
    * 入力データのサニタイズを行い、XSS攻撃を防御します。
    * APIアクセス制御により、不正なアクセスを防御します。
* **パフォーマンス:**
    * データベースのインデックスを最適化することで、検索速度を向上させました。
    * 画像の遅延読み込みにより、初期表示速度を向上させました。
    * APIレスポンスのキャッシュにより、サーバー負荷を軽減します。
* **テスト:**
    * ユニットテスト、結合テスト、E2Eテストを実施し、品質を確保しています。
    * Postmanを用いて、APIエンドポイントのテストを行っています。

## 苦労した点

* スクレイピングの安定性を確保するために、サイトの構造変更に柔軟に対応する必要がありました。
* 様々な画面サイズで最適なレイアウトを提供するために、レスポンシブデザインの実装に時間を要しました。
* 大量の求人情報を効率的に管理するためのデータベース設計に苦労しました。

## 今後の展望

* 求人情報のフィルタリング・検索機能を強化します。
* 応募企業情報の分析機能を追加します（選考通過率、応募傾向など）。
* 他の求人サイトのスクレイピングにも対応する予定です。
* ユーザー認証機能を追加し、個人ごとのデータ管理を可能にします。
* 応募企業とのチャット機能を追加し、コミュニケーションを円滑にします。
* AIによる応募書類の添削機能を検討しています。
